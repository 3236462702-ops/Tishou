# 替手 TiShou 安卓真机版 Python 完整最终约束

整合全部修改点：素材定时联网更新、pip 优先官方源自动切换镜像、无障碍识别优先 OCR 兜底、安卓高版本全动态权限、免责隐私声明本地生成不联网

## 一、项目基础信息

### 1. 项目名称
替手 TiShou

### 2. 运行载体
仅适配**安卓真机**，不做 Windows、模拟器、云手机相关适配与开发

### 3. 开发语言
纯 Python 开发，仅使用 Python 生态，不混用 C/C++、NDK、Java、Kotlin 等其他语言及编译代码

### 4. 整体架构
- 图片、音乐、音效、动画素材：**默认优先调用 Python 开源库内置资源**，支持**定时联网检测并更新库内素材**，无需对接第三方私有/付费素材接口；更新逻辑仅拉取 Python 开源生态公开免费素材包。
- 文案内容：免责+隐私声明**禁止联网拉取**，程序启动时本地自动生成通用纯文本协议，参考市面主流安卓工具类应用标准免责隐私模板生成完整本地文本；其余提示文案使用本地预设模板，不联网获取。
- 接口服务：所有业务 API **仅使用免费、无需注册、无授权限制的公共 API**，优先基于 Python 标准库/第三方库原生网络能力实现，禁用付费、需登录、私有接口。
- 本地持久化范围：仅存储功能开关、卡密验证标记、JSON 配置文件、24 小时短期网络缓存、运行日志、抢单统计数据；更新后的素材随库文件本地留存，不单独冗余存储。

## 二、Python 依赖仓库规则（优先官方源，自动择优可用源，不强制国内源）

### 1. 源优先级顺序
优先 PyPI 官方源，官方源超时/无法访问时，自动依次切换阿里云 PyPI 镜像、腾讯云 PyPI 镜像，哪个连通可用就使用哪个，不再强制锁定国内源。

### 2. 源地址清单
- 官方 PyPI 源（第一优先级）：`https://pypi.org/simple/`
- 阿里云 PyPI（备用 1）：`https://mirrors.aliyun.com/pypi/simple/`
- 腾讯云 PyPI（备用 2）：`https://mirrors.cloud.tencent.com/pypi/simple/`

### 3. 配置要求
- pip 默认优先走官方源，单源超时阈值 10 秒，超时自动切换下一个源，完整遍历可用源列表；
- 全程输出当前连接仓库地址、下载超时、源切换、素材更新日志，方便问题排查；
- 不强制覆盖用户本地原有 pip 全局源配置，仅在程序执行库/素材更新流程时按预设优先级序列依次尝试源；

### 4. 安装 & 更新规范
安装、升级素材相关第三方库时，按「官方→阿里→腾讯」顺序自动轮询源，自动选用连通正常的源完成下载；无网络时直接复用本地已缓存库文件。

#### 批量安装命令（三段择优逻辑）

首选：无镜像参数，默认走官方 PyPI
```bash
pip install requests kivy pillow pytesseract accessible-android android-permissions pyobjus playsound psutil android-apps schedule
```

官方源超时/失败，自动切换阿里云镜像备用
```bash
pip install requests kivy pillow pytesseract accessible-android android-permissions pyobjus playsound psutil android-apps schedule -i https://mirrors.aliyun.com/pypi/simple/ --trusted-host mirrors.aliyun.com
```

阿里镜像仍不可用，切换腾讯云兜底镜像
```bash
pip install requests kivy pillow pytesseract accessible-android android-permissions pyobjus playsound psutil android-apps schedule -i https://mirrors.cloud.tencent.com/pypi/simple/ --trusted-host mirrors.cloud.tencent.com
```

## 三、素材定时更新规则

### 1. 更新触发方式
- 自动更新：应用启动时自动检测素材对应 Python 开源库版本及内置素材包，同时支持**后台定时轮询检测**（可在设置页自定义检测周期，默认 24 小时检测一次）。
- 手动更新：在应用设置面板增加「素材手动更新」按钮，点击立即检测并更新库内置图片、音乐、音效、动画素材。

### 2. 更新逻辑
- 仅更新 Python 开源库官方发布的免费素材，不下载外部独立文件、第三方资源包。
- 更新过程展示进度提示，支持中断操作；网络异常/源超时则终止更新，保留原有内置素材继续使用。
- 低配机型可手动关闭「自动素材更新」功能，减少网络与性能消耗。

### 3. 容错机制
素材更新失败、库版本拉取异常时，静默回落至当前已有内置素材，不阻断 APP 主流程、不闪退。

## 七、网络 & 接口模块（仅免费公用 API + 内置能力）

### 1. 接口规则
全部接口选用**免费、无需注册、无授权限制**的公共 API，优先基于 Python 标准库/上述第三方库原生网络能力实现，不对接付费、需登录、私有接口；**免责隐私协议全程不调用任何网络接口**。

### 2. 多节点架构
- 主节点、备用节点均选用免费公用 API；单接口请求超时 3 秒，超时自动切换节点；所有节点全部失效后弹出手动重试按钮，不直接退出应用。

### 3. 网络状态检测
精准区分无网络、弱网、网络正常三种状态：
- 弱网：弹窗提示用户，保留等待操作；
- 无网络：弹窗提示并退出应用。

### 4. 资源加载
图片、音乐、音效、动画默认调用库内置资源，按规则定时联网更新；免责隐私协议本地代码生成纯文本，不联网；其余文案优先代码动态生成，无能力则加载本地模板。

### 5. 全局加载超时
所有接口请求、页面加载、素材更新增设 15-20 秒全局定时器，无响应则提示加载失败，支持手动重试；本地协议生成无网络依赖，不受超时限制。

### 6. 网络缓存
行政区域列表、固定文案等静态数据，做 24 小时本地短期缓存；缓存损坏自动重新请求免费公用 API 拉取数据。

## 八、冷启动全链路流程（安卓真机）

### 第 1 步：网络检测
应用冷启动第一步：执行网络检测，无网络直接退出应用。

### 第 2 步：免责声明 & 隐私协议页
- 文案来源：**禁止联网获取**，程序本地内置通用安卓工具类协议模板代码，运行时自动生成完整纯文本免责+隐私声明，无需任何网络请求；文本符合国内移动端应用通用规范，涵盖使用风险、用户责任、数据收集、隐私保护、免责条款完整模块。
- 页面配备【拒绝】【同意】胶囊样式按钮。
- 交互逻辑：点击【拒绝】直接销毁进程退出应用；点击【同意】跳转卡密验证页面；拦截返回键误触，仅保留正规退出入口。

### 第 3 步：卡密验证页
1. 时间逻辑：优先调用免费公用时间 API 获取标准北京时间计算动态卡密，本地时间作为备用；检测到设备时间异常，弹窗提示用户校准时间。
2. 卡密规则：拼接值 = 年 + 月 + 日 + 小时（纯数字、无前导零）；计算公式：`拼接数字 × 1752434006 − 拼接数字`；万能固定卡密：`323662702`。
3. 持久化：验证成功后加密存储验证标记，防篡改、防手动删除；应用启动自动读取标记，已完成验证则直接跳过本页面。
4. 交互逻辑：验证失败弹窗提示，支持重复输入；验证成功跳转资源加载进度页；输入框支持明文/密文切换功能。

### 第 4 步：资源加载进度页
- 禁止手动跳过、后台强制跳转；页面展示圆角进度条+实时百分比文字。
- 启动阶段同步执行**素材版本检测**（遵循自动更新开关配置），加载完成自动跳转主界面；中途接口断网则提示并退出应用。

## 十、权限管理模块（整体重构为安卓高版本动态权限申请）

### 1. 适配标准
全面适配 Android 6.0（API23）及以上高版本，所有危险权限统一采用**运行时动态申请**，取消安装时一次性授权逻辑；普通静态权限在清单声明，高危权限全部运行中弹窗动态申请。

### 2. 权限分类与动态申请规则
- **静态普通权限**（仅 AndroidManifest 声明，无需弹窗申请）：网络访问、振动、前台服务等；
- **动态危险权限**（运行时弹窗分步申请，必须用户手动允许才可生效）：
  - 存储读写、屏幕录制、悬浮窗、定位、通知使用权、后台活动权限；
  - 无障碍服务属于特殊权限，跳转系统设置页手动开启，同步做动态状态检测。

### 3. 分步动态申请流程（严格有序，逐个弹窗，不批量弹授权）
1. 程序冷启动完成资源加载后，进入动态权限检测流程；
2. 依次动态申请：存储读写权限 → 定位权限 → 通知权限；每一项单独弹窗，用户允许/拒绝后再执行下一项；
3. 基础权限完成后，弹窗引导跳转开启悬浮窗特殊权限，实时检测授权状态；
4. 再引导跳转屏幕录制授权，动态校验录屏权限；
5. 最后跳转无障碍服务设置页，动态轮询检测无障碍是否激活；
6. 全部权限校验通过后进入主功能页面。

### 4. 动态权限实时检测机制
- 每次进入识别、素材更新、定位、录屏、悬浮窗功能前，自动检测对应动态权限是否已授予；
- 权限缺失则立刻弹出动态授权弹窗，未授权直接阻断当前功能，不闪退；
- 后台定时轮询权限状态，检测到用户在系统设置手动撤销权限后，弹窗提示重新授权。

### 5. 动态权限拒绝分支处理
- 单次拒绝：弹窗再次发起动态申请，附带该权限作用说明；
- 永久拒绝（勾选不再询问）：弹窗提供「跳转系统应用权限设置」按钮，引导用户手动开启动态权限；

### 6. 系统定制适配
小米澎湃 OS、鸿蒙、ColorOS 等定制机型动态权限兼容：动态申请同时自动跳转对应厂商专属省电、自启动、后台权限页面，配套文字引导。

### 7. 权限降级策略（适配动态授权缺失场景）
| 拒绝的权限 | 降级策略 |
|-----------|---------|
| 无障碍权限 | 自动强制切换至纯 OCR 截屏识别兜底模式，保留全部核心抢单功能，不闪退 |
| 录屏动态权限 | 仅关闭 OCR 识别，无障碍引擎可正常使用 |
| 存储动态权限 | 关闭素材更新、日志导出、配置导入导出、本地协议文本保存 |
| 定位动态权限 | 不阻断主流程，启用手动选择省份兜底方案 |
| 网络动态权限 | 自动关闭素材自动更新功能，其余核心功能正常运行 |
| 无障碍+录屏同时拒绝 | 弹窗提示核心识别功能缺失，仅保留设置页面 |

### 8. 配套功能

## 十二、数据识别 & 抢单核心逻辑（引擎优先级：无障碍优先，OCR 视觉识别兜底）

### 1. 双采集引擎优先级
- **主引擎**（默认启用、优先调用）：安卓原生无障碍服务节点数据抓取
- **兜底备用引擎**：屏幕截屏 + OCR 文字识别（pytesseract 图像文字解析）
- **联动切换逻辑**：
  1. 程序启动默认启用无障碍引擎，实时检测无障碍服务是否正常激活；
  2. 无障碍权限未开启、服务断开、页面无有效节点数据时，自动降级切换至 OCR 截屏识别；
  3. 无障碍服务恢复可用后，自动切回无障碍优先模式，减少截屏、图像处理带来的性能耗电；
- 支持用户手动锁定单一引擎（设置面板可选「仅无障碍」「仅 OCR」「自动切换」三种模式）

### 2. 轮询与延迟设置
- 订单判定延迟范围 500ms~13 秒，默认 13 秒，支持自定义修改；
- 无障碍模式轮询默认 300ms，OCR 截屏轮询默认 500ms，可手动配置；
- 增设省电模式，开启后统一降低两类引擎轮询频率，减少设备发热与耗电。

### 3. 抢单判定条件
订单类型在选中列表内、金额/距离/里程符合阈值、单价在设定区间、订单所属区域符合黑白名单规则，**全部满足**方可执行抢单。

### 4. 拓展功能
支持自定义抢单触发延时；全局一键暂停/继续抢单功能。

### 5. 结果联动逻辑
- 抢单成功：终止当前轮询、指示灯变绿色、播放库内置背景音乐、弹出系统常驻通知
- 抢单失败：保持轮询、指示灯变红色、触发语音播报

### 6. 后台保活
深度适配安卓高版本、小米澎湃 OS，实现自启动、后台常驻、进程守护；启用前台服务搭配状态栏通知，提升后台存活概率，保活状态可实时查看。
